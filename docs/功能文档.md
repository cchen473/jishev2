# NebulaGuard 地震指挥系统功能文档

## 1. 文档目的

本文档描述当前 NebulaGuard 的已实现能力（Earthquake Mode），用于研发、测试和演示对齐。

## 2. 产品定位

系统已重构为单一地震场景，基准城市为成都。

- 基准坐标：`30.5728, 104.0668`
- 目标：社区级震情上报、避险分析、群组广播、指挥协同

## 3. 功能总览

### 3.1 用户与社区

1. 用户注册：用户名、密码、显示名称、社区名称。
2. 用户登录：签发 Bearer Token。
3. 社区自动化：注册时自动“创建或加入”同名社区。
4. 当前用户查询：返回用户信息与所属社区。

### 3.2 移动端震情上报

入口：`/mobile`

支持上报字段：

1. 经纬度
2. 震感等级（1-12）
3. 建筑/房屋结构类型
4. 周边构造补充说明
5. 现场描述
6. 现场图片（可选）

上报后动作：

1. 持久化到 `earthquake_reports`
2. 触发 VLM/规则避险建议
3. 生成社区避险通知
4. WebSocket 广播给同社区在线用户

### 3.3 VLM 场景分析与避险建议

1. 输入：震感等级 + 建筑结构 + 现场描述 + 可选图片
2. OpenAI 可用：使用视觉模型生成 3-6 条中文可执行建议
3. OpenAI 不可用：降级为规则建议

### 3.4 社区通知

1. 自动通知：每次地震上报后自动推送社区避险通知
2. 手动通知：指挥端可编辑标题/内容并广播
3. 通知查询：支持拉取社区通知列表

### 3.5 指挥中心

入口：`/`

1. 登录/注册与社区身份显示
2. 社区范围内实时报告与通知
3. 地图态势（成都基准）
4. 多智能体任务终端（WebSocket）
5. 社区手动广播面板

## 4. 实时协议（WebSocket）

地址：`ws://localhost:8000/ws/mission?token=...`

客户端消息：

1. `{"type":"start_mission","description":"..."}`
2. `{"type":"fetch_recent_reports","limit":50}`
3. `{"type":"fetch_notifications","limit":30}`
4. `{"type":"ping"}`

服务端消息：

1. `status/error/pong`
2. `field_report`（地震上报广播）
3. `community_alert`（社区通知广播）
4. `community_notifications`（通知批量拉取结果）
5. 多智能体阶段消息（任务流程）

## 5. HTTP 接口

### 5.1 认证与社区

1. `POST /auth/register`
2. `POST /auth/login`
3. `GET /auth/me`
4. `GET /community/me`
5. `GET /community/notifications`
6. `POST /community/alerts`

### 5.2 地震业务

1. `GET /reports/recent`
2. `GET /system/summary`
3. `POST /report/earthquake`
4. `POST /report/earthquake_with_media`
5. `POST /ai/route_advice`

### 5.3 指挥任务

1. `POST /mission/start`
2. `GET /missions/{mission_id}`

## 6. 数据库结构（SQLite）

新增核心表：

1. `users`
2. `communities`
3. `community_memberships`
4. `earthquake_reports`
5. `community_notifications`

保留表：

1. `missions`
2. `mission_events`

## 7. 配置项

文件：`backend/.env`

关键变量：

1. `BASE_CITY`、`BASE_LAT`、`BASE_LNG`
2. `OPENAI_API_KEY`、`OPENAI_MODEL`、`OPENAI_VLM_MODEL`
3. `AUTH_SECRET`、`AUTH_TOKEN_EXP_MINUTES`
4. `DATABASE_PATH`、`UPLOAD_DIR`、`MAX_UPLOAD_MB`

文件：`frontend/.env.local`

1. `NEXT_PUBLIC_BACKEND_HTTP_URL`
2. `NEXT_PUBLIC_BACKEND_WS_URL`
3. `NEXT_PUBLIC_BACKEND_PORT`
4. `NEXT_PUBLIC_MAPBOX_TOKEN`

## 8. 异常与降级

1. 无 OpenAI Key 时，VLM建议自动降级为规则建议（`analysis_status=mock`）。
2. Token 无效或缺失时，受保护接口返回 401。
3. 图片非 `image/*` 或超过大小限制时，上报返回 400/413。
4. WebSocket 异常时，前端自动重连。

## 9. 快速验收

1. 注册新用户并加入社区成功。
2. 登录后可查看社区报告和系统摘要。
3. 移动端可上报震感、建筑结构和图片。
4. 上报后可收到 `shelter_advice` 和 `community_alert`。
5. 指挥中心可手动发送社区广播。
