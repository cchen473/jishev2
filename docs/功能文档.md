# NebulaGuard 地震指挥系统功能文档

## 1. 产品定位

NebulaGuard 是面向社区级地震应急的协同指挥系统，目标不是“聊天演示”，而是打通：

**图像研判 -> 自动调度 -> 事件工单 -> 社区执行 -> 回执复盘**。

城市与社区范围可配置，不限定单一城市。

---

## 2. 功能全景

## 2.1 账号与社区

- 用户注册/登录
- 自动创建或加入社区
- 社区维度的数据隔离

## 2.2 地震上报与通知

- 上报震感、建筑结构、现场描述、图片
- 自动生成避险通知
- 管理端手动广播与回执确认

## 2.3 地震 VLM 受灾搜救（主链路）

- 上传鸟瞰图后调用 VLM 做受灾目标识别
- 输出结构化 victims/routes/notes
- 后端生成标注图 URL（原图 + 标注图）
- 返回算法指标：复杂度、覆盖率、热点聚类

## 2.4 自动调度 Agent

- 每次搜救分析完成后自动触发
- 自动创建/更新 incident、task、dispatch
- 幂等防重、上限保护、失败降级
- 审计日志与时间轴可追溯

## 2.5 指挥闭环能力

- 事件中心（Incident）
- 任务看板（Task）
- 队伍编组（Team）
- 资源调度（Dispatch）
- 居民报平安（Checkin）
- 失联管理、避难点、风险区、道路阻断

---

## 3. 管理端交互结构（/）

左侧工作区导航（多页分区）:

1. **总览**：地图态势 + 指标 + 算法评估
2. **调度**：事件/任务 + Agent 执行轨迹
3. **社区**：通知回执 + 群聊协同

右侧操作区（全局可操作）:

- 指挥终端
- 社区群聊
- 地震搜救

关键体验改造：右侧区与聊天区独立滚动，避免整页被撑长。

## 6. 实用价值（为什么这个项目有价值）

## 6.1 对指挥者

- 减少“看图靠经验”的不确定性：VLM + 标注图可解释。
- 自动调度减少首轮响应时间。
- 事件与任务可追踪，避免“口头命令丢失”。

## 6.2 对社区执行

- 通知、群聊、回执在同一系统闭环。
- 居民状态（平安/需救援）可量化，不再靠人工统计。

## 6.3 对复盘与治理

- 自动动作有审计与时间轴，便于复盘问责。
- 可扩展为演练模式、评分系统和 SOP 管理。
