# NebulaGuard 双角色用户画像与使用说明

## 1. 适用对象

本系统面向两类核心用户：

1. **指挥者（社区管理端）**：负责态势研判、工单调度、通知下发、回执跟踪。  
2. **居民（用户端）**：负责现场上报、上传图片、接收避险建议、在社区群聊协同反馈。

---

## 2. 用户画像

## 2.1 指挥者画像

- 典型身份：街道值班员、社区网格员、应急联络员。  
- 主要目标：快速建立事件、分派任务、确认执行状态。  
- 关注指标：上报总量、活跃任务、待救援人数、通知回执率。  
- 操作习惯：桌面端集中处理、多模块切换、偏数据化视图。

## 2.2 居民画像

- 典型身份：社区住户、楼栋长、志愿者。  
- 主要目标：快速上报现场、上传周边图片、获得可执行避险建议。  
- 关注信息：当前是否安全、家人状态、附近风险、社区最新通知。  
- 操作习惯：移动端快速提交，步骤尽量少。

---

## 3. 指挥者如何使用（Web 主站 `/`）

## 3.1 页面分区含义

左侧工作区：

1. **总览**：地图态势、核心指标、近期调度与时间轴。  
2. **调度**：事件中心、任务看板、自动调度执行轨迹。  
3. **社区**：通知回执、社区群聊摘录。

右侧滑出工作台：

1. **指挥终端**：任务流与通知发送。  
2. **社区群聊**：实时协同沟通。  
3. **地震搜救**：上传现场图并获取搜救分析。

## 3.2 推荐操作路径

1. 在 `地震搜救` 上传图像并提交分析。  
2. 切到 `调度`，检查自动创建的事件与任务。  
3. 在 `社区` 下发通知并跟踪已读/确认。  
4. 在 `总览` 通过地图与时间轴复核当前态势。

---

## 4. 居民如何使用（移动端 Web `/mobile` 或 Flutter）

## 4.1 页面区域含义

1. **状态区**：当前登录态、主站登录入口。  
2. **上报表单区**：经纬度、震感等级、建筑类型、结构说明、现场描述、图片上传。  
3. **建议结果区**：系统给出的避险建议、上报摘要、现场图链接。

## 4.2 推荐操作路径

1. 先登录账号（与主站共用账号体系）。  
2. 打开 `/mobile`，填写现场信息，优先上传现场图片。  
3. 提交后按建议结果先行避险，再在社区群聊同步状态。  
4. 环境变化后再次上报，保持信息持续更新。

---

## 5. 角色协同价值

1. 居民提供一线图像与现场描述。  
2. 指挥者在管理端完成调度与广播。  
3. 通知回执与聊天反馈形成执行闭环。  
4. 时间轴沉淀过程数据，便于后续复盘。
