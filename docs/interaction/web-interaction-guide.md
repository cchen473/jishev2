# NebulaGuard Web 交互指南（管理端）

角色视角与居民端流程详见：`docs/interaction/dual-persona-usage-guide.md`。

## 1. 这套页面怎么用

登录后左侧有 3 个工作区按钮：

1. **总览**：看地图、关键指标、算法评估和热点聚类。
2. **调度**：看事件板、任务看板、Agent 自动执行轨迹。
3. **社区**：看通知、回执、群聊消息，做协同指挥。

> 这 3 个工作区相当于“多页分区”，不是所有内容挤在一个长页面里。

---

## 2. 一次完整操作流程（建议）

### 步骤 A：发起地震搜救分析

1. 右侧切到 `地震搜救` 面板。
2. 上传 1~6 张现场图，填写描述与坐标。
3. 提交后查看：
   - 受灾目标列表
   - 标注图链接
   - 搜索/救援路线
   - 算法指标（复杂度、覆盖率、热点）

### 步骤 B：确认 Agent 自动调度结果

1. 切到左侧 `调度` 工作区。
2. 在 `Agent 执行轨迹` 看本次自动动作：
   - 新建了多少事件
   - 新建了多少任务
   - 新建了多少调度记录
3. 在 `事件中心` / `任务看板` 做人工复核与推进。

### 步骤 C：社区执行闭环

1. 切到左侧 `社区` 工作区。
2. 在通知区发送或确认通知。
3. 在群聊区同步 AI 建议和现场反馈。
4. 使用通知回执确认是否真正触达用户。

---

## 3. 右侧面板说明

右侧是持续可操作区，包含 3 个功能 Tab：

- **指挥终端**：任务流消息 + 手动广播
- **社区群聊**：居民消息、AI 跟进、单独提问
- **地震搜救**：图像分析入口与结果展示

设计原则：右侧内容独立滚动，不再把整页无限拉长。

---

## 4. 你会在前端看到 Agent 做了什么

可见位置：

1. `自动调度 Agent` 卡片：显示最近执行状态、生成数量、规划器来源。
2. `调度` 工作区中的 `Agent 执行轨迹`：按时间列出每次自动执行的详细结果。
3. `复盘时间轴`：记录自动调度执行事件，可追溯。

---

## 5. 常见问题

### Q1. 看到 Unsupported command / Not Found

通常是前端新、后端旧进程。重启后端并强刷页面。

### Q2. 看不到自动调度记录

需要先至少完成一次 `地震搜救分析`，自动调度才会触发。

### Q3. 页面左右高度不一致

新版已改为固定视口 + 内部滚动；若仍异常，先清缓存再刷新。

---

## 6. 快速自检清单

1. 上传图像后能看到分析结果和标注图链接。
2. 自动调度记录中出现新的 run。
3. 事件/任务列表有 `AI-AUTO` 来源项。
4. 社区通知可提交回执。
5. 群聊区可持续滚动且输入区固定可见。
