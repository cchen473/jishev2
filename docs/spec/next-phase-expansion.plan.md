# NebulaGuard 后续功能扩展方案（评审稿）

## 目标

在现有“地震 VLM + 自动调度闭环”基础上，继续提升到可持续指挥体系：

1. 提升态势认知（风险热力、覆盖率、负载）。
2. 提升调度质量（冲突检测、审批与回滚）。
3. 提升居民侧触达（多渠道通知与回执闭环）。
4. 提升复盘能力（时间轴自动报告）。

## 功能包（建议按批次）

### Batch A：调度安全增强

- 自动调度审批开关（全自动/人工确认）。
- 按 run_id 回滚自动调度结果。
- 队伍冲突检测与负载均衡建议。

### Batch B：态势与地图增强

- 余震风险分层热力图。
- 楼栋级搜救覆盖率看板。
- 风险区 + 道路阻断 + 路线联动展示。

### Batch C：居民触达增强

- 多渠道通知（App/短信/语音）统一回执。
- 失联与报平安自动对账。
- 避难点拥挤度预测与分流建议。

### Batch D：复盘与演练

- 演练模式（沙盘回放 + 评分）。
- SOP 模板一键下发。
- 指挥复盘自动报告（Markdown/PDF）。

## UI 方向（沿用 ui-ux-pro-max）

- 风格：Data-Dense + 深色奢简 + 冷金强调。
- 信息架构：地图态势 / 工单看板 / 时间轴三栏联动。
- 动效：150~250ms，禁止炫技动画。
- 可读性优先：高密度但分组清晰，关键状态一眼可判。

## 建议里程碑

- `v2.3`: 调度安全增强（审批+回滚+冲突检测）
- `v2.4`: 态势地图增强（热力+覆盖率+路线联动）
- `v2.5`: 居民触达增强（多渠道+对账+分流）
- `v3.0`: 复盘演练体系（自动报告+评分）
