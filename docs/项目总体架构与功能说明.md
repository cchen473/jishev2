# NebulaGuard 项目总体架构与功能说明

## 1. 项目定位

NebulaGuard 是面向地震应急场景的社区级智能指挥与协同系统。  
系统目标不是做单点 AI 演示，而是打通完整业务闭环：

**居民上报 -> AI 研判 -> 自动调度 -> 多端协同 -> 预警触达 -> 复盘审计**

系统同时服务两类核心用户：
- 指挥端（Web）：社区管理者、应急调度者
- 居民端（Flutter）：社区居民、志愿协同用户

---

## 2. 总体技术架构

### 2.1 架构分层

- 前端层：Next.js 指挥台 + Flutter 移动端
- 服务层：FastAPI 统一 REST 与 WebSocket
- AI 层：VLM 图像理解 + 自动调度 Agent
- 数据层：SQLite 事件化存储
- 实时层：WebSocket 广播关键状态变更

### 2.2 核心数据流

1. 居民提交地震上报（文字、位置、图片）
2. 后端校验图片并触发 VLM 分析
3. 生成结构化建议（风险摘要、行动建议、关注目标）
4. 自动调度 Agent 读取队伍与任务状态并生成派单
5. 任务、通知、事件实时推送到管理端与移动端
6. 全链路写入审计日志与复盘时间轴

---

## 3. 实现技术路线

### 3.1 前后端技术栈

- Web：Next.js + React + Tailwind
- Mobile：Flutter
- Backend：FastAPI + Uvicorn
- DB：SQLite（一期）
- 实时通信：WebSocket

### 3.2 AI 与调度路线

- 图像理解：通过 VLM 读取居民上传现场图像，输出可执行建议
- 调度执行：Agent 基于规则约束和当前资源生成任务
- 审计保障：自动化动作记录为可追溯事件

### 3.3 工程目标

- 保持 API 向后兼容
- 在弱网环境下保证核心链路可用
- 支持演示场景下快速重置业务数据（保留账号）

## 4. 功能模块说明

## 4.1 账号与社区管理

- 用户注册、登录、身份鉴权
- 社区归属与社区数据隔离
- 角色基础能力区分

## 4.2 地震上报

- 提交震感、描述、位置与现场图片
- 支持围绕建筑结构与周边环境上报
- 上报内容进入后续分析与调度链路

## 4.3 AI 助手与图像研判

- 对现场图像进行语义理解
- 返回结构化躲避建议与行动建议
- 结果支持 Markdown 渲染，增强可读性

## 4.4 自动调度 Agent

- 分析完成后自动触发调度
- 根据队伍、资源、事件优先级生成任务
- 输出状态、数量、时间、错误摘要等数据轨迹

## 4.5 事件中心与任务看板

- 事件状态流转与优先级管理
- 任务创建、派发、执行、回执
- 指挥端统一视图追踪全过程

## 4.6 社区群聊与协同

- 指挥端和居民端实时消息同步
- 支持应急信息快速回传
- 用于通知确认与现场协同

## 4.7 一键预警

- 管理端一键发送社区预警
- 居民端实时收到弹窗提醒
- 用于突发风险下的快速触达

## 4.8 地图态势

- 在地图上展示地震态势与关键点位
- 结合右侧工作区进行联动决策
- 作为指挥总览主视觉入口

## 4.9 复盘与审计

- 关键动作写入审计日志
- 时间轴回放事件演进
- 支持复盘、问责与流程优化

---

## 5. 用户画像与使用价值

## 5.1 指挥者画像

- 人群：社区管理者、应急值守人员
- 核心诉求：快速看清态势、稳定派发任务、追踪执行结果
- 获得价值：统一指挥界面 + 自动调度 + 回执闭环

## 5.2 居民画像

- 人群：社区住户与家庭成员
- 核心诉求：灾时快速获得可执行建议
- 获得价值：低门槛上报 + AI 躲避建议 + 实时预警

## 5.3 协同执行画像

- 人群：志愿者与救援队成员
- 核心诉求：任务清晰、状态同步、沟通成本低
- 获得价值：结构化任务 + 实时更新 + 可回执执行

---

## 6. 项目特色

1. 闭环化：从信息采集到执行复盘的一体化链路
2. 双端协同：同时覆盖管理端与居民端真实场景
3. 自动化可控：Agent 自动执行但全程可追踪
4. 实战导向：强调可用性、时效性和稳定性
5. 可扩展：可继续扩展多模型、多数据库与多渠道通知

## 7. 演示字幕配文（按功能模块）

- 项目总览：此系统面向地震应急协同，基于 Web、Mobile、AI 与实时通信构建“上报到调度”的闭环平台。
- 注册登录：此功能用于建立可信身份体系，基于认证鉴权与社区归属模型实现多角色协同。
- 地震上报：此功能用于采集一线灾情，基于结构化表单与图片上传实现高质量现场输入。
- 图像研判：此功能用于将现场图像转化为行动建议，基于视觉语言模型实现灾情理解与辅助决策。
- 自动调度：此功能用于减少人工派单延迟，基于规则约束与资源状态实现任务自动生成与分配。
- 事件看板：此功能用于统一处置流程，基于事件与任务状态流转实现全过程可视化管理。
- 社区群聊：此功能用于提升多方协同效率，基于 WebSocket 实现实时消息同步与反馈。
- 一键预警：此功能用于缩短风险触达链路，基于后端广播和移动端监听实现秒级通知下发。
- 地图态势：此功能用于增强空间决策能力，基于地图可视化实现风险区域与处置状态联动展示。
- 复盘审计：此功能用于保障可追溯治理，基于审计日志与时间轴实现过程回放与复盘分析。
